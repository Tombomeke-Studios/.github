## About

**Tombomeke Studios** is a technical software organisation founded by [Tom Dekoning](https://github.com/tombomeke-ehb), a second-year Applied Informatics student at *Erasmus University College Brussels*. The organisation serves as an engineering ecosystem for building production-grade software — from AI-powered platforms and .NET frameworks to developer tooling published on NuGet.

> *"We don't just write code — we architect systems that last."*

Every project in this organisation is driven by a commitment to **Clean Architecture**, **SOLID principles**, and real-world applicability. Whether it's automating documentation workflows, analysing contracts with AI, or providing a modular game framework as a design reference, the goal is always the same: software that is maintainable, extensible, and built to production standards.

## Engineering Principles

The following standards are non-negotiable in every project under this organisation:

| Principle | Description |
|---|---|
| **Clean Code & SOLID** | Readability and maintainability are treated as architectural requirements, not afterthoughts. |
| **Modular Decoupling** | Systems are decomposed into independent, reusable components — monoliths are avoided by design. |
| **Type Safety** | TypeScript enforces correctness at the frontend; strongly-typed C# eliminates ambiguity in the backend. |
| **Automated Pipelines** | Every project ships with CI/CD via GitHub Actions for build validation, testing, and distribution. |
| **Documentation First** | Docs are written alongside code — never as a post-release task. |
| **Continuous Innovation** | Actively tracking .NET and JavaScript ecosystem releases, cloud-native patterns, and AI tooling. |
